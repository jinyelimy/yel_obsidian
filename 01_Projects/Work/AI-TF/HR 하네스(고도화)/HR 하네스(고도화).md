# HR 하네스(고도화)

> `ehr-harness-plugin`을 바탕으로 연말정산 도메인용 HR 하네스를 이해하고, 신입 개발자도 따라갈 수 있는 순서로 분석·설계·구현·테스트를 정리하는 작업 공간입니다.

## 🎯 목표
- 원본 플러그인이 무엇을 만들어주는지 먼저 눈으로 이해한다.
- 그다음 원본 플러그인이 어떤 구조로 동작하는지 분석한다.
- 연말정산 도메인을 넣기 위해 무엇이 부족한지 찾고, 작은 단위부터 설계·구현·테스트한다.
- 최종적으로 내 Git 저장소에 올릴 수 있는 하네스 구조와 릴리즈 기준을 만든다.

## 📌 현재 상태
- `ehr-harness-plugin` 레포 접근 권한 확보
- 기존 구조를 신입 개발자 학습 순서 기준으로 재정리함
- 다음 집중 구간은 `결과물 투어 -> 원본 구조 분석 -> gap 분석` 순서

## 📁 폴더 구조
- `00_읽는순서`: 왜 이 순서로 보는지, 각 단계에서 무엇을 확인해야 하는지
- `01_결과물투어`: 하네스가 프로젝트에 적용되면 무엇이 생기고 어떻게 보이는지
- `02_원본구조분석`: 원본 플러그인이 어떤 구조로 만들어졌는지
- `03_Gap분석`: 연말정산 도메인을 넣기 전에 부족한 점이 무엇인지
- `04_도입설계`: 부족한 점을 채우기 위한 하네스 구조 설계
- `05_첫Slice`: 가장 작은 기능 단위로 구현을 시작하는 계획
- `06_테스트`: 구현한 slice를 어떻게 확인할지에 대한 기준
- `07_릴리즈`: 최종 정리, 버전 전략, 배포 전 체크리스트
- `logs`: 날짜별 작업 기록
