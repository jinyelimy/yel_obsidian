# 첫 수직 slice 구현 계획

## 왜 slice로 가나
- 연말정산 전체를 한 번에 넣으면 신입 개발자가 어디서 막혔는지 찾기 어렵다.
- 그래서 가장 얇은 단위 하나를 끝까지 붙여보고, 그 결과를 기준으로 다음 범위를 넓힌다.

## 첫 slice 기준
- 기능 1개
- 연도 1개
- 질문 패턴 1~2개
- reference 1개 이상
- 필요하면 skill 1개까지

## 추천 범위 예시
- `2025년 의료비`처럼 연도와 기능이 모두 고정된 질문 묶음
- 또는 `연말정산 특정 화면 1개`와 연결된 테이블/프로시저 확인 흐름

## 구현 순서
1. 대상 기능을 하나 고른다.
2. 해당 기능의 화면, 테이블, 프로시저, 배치 흐름을 최소한으로 정리한다.
3. 이것이 `reference`인지 `skill`인지 먼저 나눈다.
4. 하네스 안에 실제 문서/규칙을 추가한다.
5. 바로 테스트 질문으로 검증한다.

## 완료 조건
- 같은 질문을 다시 받았을 때, 하네스가 어디를 봐야 할지 일관되게 안내할 수 있다.
- 필요한 참조 문서가 빠지지 않았다.
- 다음 slice를 추가할 때 같은 틀을 재사용할 수 있다.
