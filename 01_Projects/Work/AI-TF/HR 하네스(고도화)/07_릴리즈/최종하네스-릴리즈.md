# 최종 하네스 릴리즈

## 릴리즈 목표
- 내 Git 저장소에 연말정산 업무에 맞게 재구성한 최종 하네스를 올린다.
- upstream 이해 문서, 연말정산 분석 문서, 실제 하네스 자산이 같이 남게 한다.

## 릴리즈 전 체크리스트
- [ ] upstream 구조와 내 구조 차이를 설명할 수 있다.
- [ ] 연말정산 전용 reference / skill / agent가 정리되어 있다.
- [ ] 안전장치(hooks, read-only, release review 기준) 포함 여부를 확인했다.
- [ ] README에 사용 목적, 적용 범위, 업데이트 방식이 정리되어 있다.
- [ ] 내 저장소에 올릴 최소 디렉토리 구조가 확정되었다.

## 버전 전략
- `v0.x`: 구조 정리와 연말정산 분석 자산 반영 단계
- `v1.0`: 실제 연말정산 소스 기준으로 반복 사용 가능한 하네스 첫 정식판
- upstream 변경이 생기면 `upstream diff -> 내 구조 영향 -> 반영 여부` 순으로 검토한다.

## 배포 시 남길 문서
- 구조 설명서
- 연말정산 소스 분석 요약
- 사용 방법 / 업데이트 방법
- 검증 / 릴리즈 체크리스트

## 배포 후 확인할 점
- [ ] 실제 연말정산 작업에서 1회 이상 재사용 가능했는지 확인
- [ ] 규칙이 너무 길거나 추상적이면 다시 skill / reference 분리를 조정
- [ ] upstream 업데이트를 따라갈 기준 문서를 남겼는지 확인
