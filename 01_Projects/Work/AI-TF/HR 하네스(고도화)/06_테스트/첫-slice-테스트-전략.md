# 첫 수직 slice 테스트 전략

## 테스트 목표
- 추가한 연말정산 지식이 실제 질문 대응에 도움이 되는지 확인한다.
- 잘못된 구조를 큰 범위로 복사하기 전에 작은 범위에서 막는다.

## 무엇을 확인하나
- 질문을 받았을 때 올바른 `reference`나 `skill`로 연결되는가
- 화면, 테이블, 프로시저, 배치 중 빠진 축이 없는가
- 설명이 너무 추상적이지 않고 실제 확인 포인트를 주는가

## 기본 테스트 방식
1. slice 범위에 맞는 질문 3~5개를 만든다.
2. 하네스를 적용한 뒤 그 질문에 답할 때 참조 경로가 일관된지 본다.
3. 빠진 정보나 과도한 정보가 있으면 문서 배치를 다시 조정한다.

## 질문 예시 템플릿
- "이 기능은 어디부터 봐야 해?"
- "이 화면 뒤에서 어떤 프로시저가 동작해?"
- "이 값이 바뀌면 영향 받는 테이블은 뭐야?"

## 통과 기준
- 질문마다 확인 시작점이 분명하다.
- 답변에 필요한 핵심 객체가 빠지지 않는다.
- 같은 유형 질문에 대해 답변 톤과 흐름이 크게 흔들리지 않는다.

## 다음 단계
- 첫 slice 테스트가 통과하면 기능을 하나 더 늘리거나, 같은 기능의 연도를 늘린다.
- 실패하면 범위를 더 줄여서 다시 붙인다.
