# 연말정산 도입 설계

## 설계 목표
- 원본 플러그인의 `shared / profile / lib` 분리를 유지한다.
- 연말정산 분석 결과가 어디에 반영되는지 한눈에 보이게 만든다.
- 신입 개발자도 "어디에 무엇을 넣어야 하는지" 판단할 수 있게 만든다.

## 설계 원칙
- `shared`에는 재사용 가능한 안전장치와 공통 커맨드만 둔다.
- 연말정산 전용 내용은 별도 프로필 또는 별도 reference/skill/agent 축으로 분리한다.
- 문서만 늘리지 말고, 실제 반복 질문을 줄일 수 있는 구조만 남긴다.
- 테스트와 audit 포인트가 빠지면 하네스 품질이 흔들리므로 관련 축은 유지한다.

## 연말정산 소스 반영 위치
- `reference/`: 화면, 프로시저, 테이블, 배치 플로우처럼 사람이 찾아보는 지식
- `skills/`: 반복 질문에 대해 일관된 방식으로 탐색/정리/응답하게 하는 규칙
- `agents/`: reviewer, tracer, impact 분석처럼 역할이 분리될 때의 판단 기준
- `docs/yearend-source-analysis.md`: 분석 근거와 설계 이유를 설명하는 문서

## 신입 기준 판단법
### `reference`에 둘 것
- 자주 찾아보는 사실
- 구조 설명
- 플로우 요약

### `skill`에 둘 것
- 반복되는 질문 패턴
- 탐색 순서
- 확인 체크리스트

### `agent`에 둘 것
- 역할이 분리되어야 하는 작업
- 리뷰 관점이 필요한 작업
- 영향도 분석처럼 별도 책임이 있는 작업
