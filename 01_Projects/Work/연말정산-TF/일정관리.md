# 일정관리

> 연말정산 TF의 교육, 회의, 인수인계 일정을 날짜 기준으로 관리하는 문서입니다.

## 사용 원칙
- 날짜가 생긴 일정은 먼저 `일정 대장`에 등록한다.
- 회의나 인수인계처럼 내용이 길어질 일정은 `상세 기록`에 링크를 만든다.
- 일정이 끝나면 결과와 후속 액션을 남기고 상태를 `완료`로 바꾼다.
- 일정 변경이 생기면 기존 기록을 지우지 말고 `결과/후속 액션`에 변경 이력을 남긴다.

## 일정 대장

| 구분 | 제목 | 날짜 | 시간 | 장소/링크 | 참석자/대상 | 준비 사항 | 상세 기록 | 결과/후속 액션 | 상태 |
|---|---|---|---|---|---|---|---|---|---|
| 교육 |  |  |  |  |  |  |  |  | 예정 |
| 회의 |  |  |  |  |  |  |  |  | 예정 |
| 인수인계 |  |  |  |  |  |  |  |  | 예정 |

## 구분 기준
- `교육`: 업무 수행을 위해 알아야 하는 제도, 시스템, 절차, 도구를 배우는 일정
- `회의`: 의사결정, 진행상황 공유, 이슈 해결을 위한 일정
- `인수인계`: 업무 범위, 자료, 권한, 처리 방식, 미해결 항목을 넘겨받거나 넘겨주는 일정

## 상태 기준
- `예정`: 일정 확정 또는 임시 등록
- `준비중`: 자료 준비, 참석자 확인, 아젠다 정리 중
- `완료`: 진행 완료 후 결과와 후속 액션 기록됨
- `변경`: 날짜, 시간, 참석자, 범위가 변경됨
- `취소`: 일정 취소됨

## 상세 기록을 따로 만드는 기준
- 결정사항이 2개 이상 있다.
- 후속 액션 담당자나 기한을 추적해야 한다.
- 인수인계 범위가 여러 항목으로 나뉜다.
- 회의록을 나중에 다시 찾아볼 가능성이 높다.
- 교육 자료, 캡처, 참고 링크가 많다.

## 상세 기록 링크 규칙
- 회의: `[[회의록/YYYY-MM-DD_회의명]]`
- 인수인계: `[[인수인계/YYYY-MM-DD_인수인계_항목명]]`
- 교육: `[[회의록/YYYY-MM-DD_교육명]]` 또는 교육 자료가 많으면 `[[자료/자료명]]`
