# 운영가이드

> 연말정산 TF 폴더를 어떻게 기록하고 관리할지 정한 운영 가이드입니다.

## 폴더 역할

```text
연말정산-TF/
├── 연말정산-TF.md              전체 대시보드 / 입구 노트
├── 업무관리.md                 받은 업무, 시작일, 종료일, 처리 결과
├── 일정관리.md                 교육, 회의, 인수인계 일정
├── 운영가이드.md               작성 규칙과 폴더 사용법
├── 회의록/                     회의와 교육의 상세 기록
├── 인수인계/                   인수인계 항목별 상세 기록
├── 자료/                       전달자료, 참고자료, 캡처 설명
└── 완료/                       끝난 업무나 오래된 정리 문서
```

## 기본 흐름

### 1. 업무를 받았을 때
1. [[업무관리]]의 `업무 대장`에 새 행을 추가한다.
2. `받은 날짜`, `요청자/출처`, `업무명`, `상태`를 먼저 채운다.
3. 바로 시작하지 않으면 상태는 `대기`로 둔다.
4. 실제 작업을 시작한 날에 `시작 날짜`를 입력하고 상태를 `진행중`으로 바꾼다.
5. 처리하면서 확인한 자료, 결정사항, 산출물은 `처리 내용`이나 `관련 링크/산출물`에 남긴다.
6. 끝나면 `종료 날짜`, `처리 내용`, `다음 액션`을 채우고 상태를 `완료`로 바꾼다.

### 2. 교육 / 회의 / 인수인계 일정이 생겼을 때
1. [[일정관리]]의 `일정 대장`에 새 행을 추가한다.
2. `구분`, `제목`, `날짜`, `시간`, `참석자/대상`, `상태`를 채운다.
3. 준비할 것이 있으면 `준비 사항`에 체크박스로 남긴다.
4. 내용이 길어질 일정은 [[회의록/README]] 또는 [[인수인계/README]] 기준에 따라 상세 문서를 만든다.
5. 일정이 끝나면 `결과/후속 액션`을 적고 상태를 `완료`로 바꾼다.

### 3. 회의 후
1. 회의 결과를 [[일정관리]]에 한 줄로 요약한다.
2. 결정사항이 많으면 `회의록/`에 상세 기록을 남긴다.
3. 해야 할 일은 체크박스로 남겨 대시보드의 `미완료 액션 아이템`에 잡히게 한다.
4. 특정 업무와 연결되면 [[업무관리]]의 해당 ID 행에도 반영한다.

### 4. 인수인계 후
1. 인수인계 범위, 받은 자료, 권한, 미해결 항목을 기록한다.
2. 넘겨받은 업무가 실제 실행 업무라면 [[업무관리]]에도 새 업무로 등록한다.
3. 미해결 항목은 담당자와 기한이 보이도록 체크박스로 남긴다.
4. 인수인계가 끝나면 [[일정관리]]의 상태를 `완료`로 바꾼다.

## 날짜 작성 규칙
- 날짜는 `YYYY-MM-DD` 형식으로 쓴다.
- 시간이 필요한 일정은 `HH:mm` 형식으로 쓴다.
- 시작 날짜는 실제로 손을 댄 날이다.
- 종료 날짜는 처리 완료, 전달 완료, 인수인계 완료 중 하나가 끝난 날이다.
- 종료되지 않은 업무는 종료 날짜를 비워둔다.

## 파일 이름 규칙
- 회의록: `YYYY-MM-DD_회의명.md`
- 교육 기록: `YYYY-MM-DD_교육명.md`
- 인수인계 기록: `YYYY-MM-DD_인수인계_항목명.md`
- 자료 메모: `자료명.md`
- 완료 보관: 원래 파일명을 유지하되 필요하면 앞에 `YYYY-MM-DD_`를 붙인다.

## 좋은 기록 기준
- 누가 봐도 `왜 이 업무가 생겼는지` 알 수 있다.
- `언제 시작했고 언제 끝났는지` 알 수 있다.
- `무엇을 확인했고 어떻게 처리했는지` 알 수 있다.
- `남은 액션과 담당자`가 보인다.
- 회의, 교육, 인수인계가 업무 기록과 연결되어 있다.

## 피해야 할 기록
- "처리함", "확인함"처럼 결과만 있고 맥락이 없는 기록
- 시작 날짜나 종료 날짜가 비어 있는 완료 업무
- 회의에서 나온 액션을 업무관리나 체크박스로 옮기지 않는 것
- 자료 파일만 저장하고 어떤 업무에 쓰였는지 설명하지 않는 것
- 변경된 일정을 기존 내용 삭제만으로 덮어쓰는 것

## 주간 정리 루틴
- [ ] [[업무관리]]에서 `대기`, `진행중`, `검토중`, `보류` 상태를 확인한다.
- [ ] 종료된 업무의 `종료 날짜`와 `처리 내용`을 확인한다.
- [ ] [[일정관리]]에서 다음 주 교육 / 회의 / 인수인계를 확인한다.
- [ ] 회의록과 인수인계 문서의 미완료 체크박스를 확인한다.
- [ ] 오래된 완료 문서는 `완료/`로 옮길지 판단한다.
- [ ] 자료가 업무나 일정과 연결되어 있는지 확인한다.

## 상태 판단 예시

| 상황 | 업무 상태 | 일정 상태 |
|---|---|---|
| 요청만 받고 아직 시작하지 않음 | 대기 | - |
| 자료 확인 중 | 진행중 | - |
| 처리 후 담당자 답변 대기 | 검토중 또는 보류 | - |
| 회의 날짜만 잡힘 | - | 예정 |
| 회의 자료 준비 중 | - | 준비중 |
| 회의가 끝나고 후속 액션 기록 완료 | - | 완료 |
| 일정이 미뤄짐 | - | 변경 |
| 업무가 더 이상 필요 없어짐 | 취소 | 취소 |
